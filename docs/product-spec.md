# 产品规格草案

## 目标

Codex 用量悬浮球用于实时查看当前 Codex 账户的剩余额度窗口，避免用户频繁进入设置页查看 rate limits。

## MVP 功能

- 读取 Codex rate limits：通过本机 `codex app-server --listen stdio://` 调用 `account/rateLimits/read`。
- 显示 5 小时窗口剩余百分比、7 天窗口剩余百分比和重置时间。
- 显示当前计划、credits 余额和是否触顶。
- 显示多用量桶，例如默认 Codex 桶和 Spark 桶。
- 手动刷新：点击刷新按钮立即读取一次。
- 自动刷新：默认每 60 秒刷新一次。
- 异常提示：Codex 未登录、CLI 不存在、读取超时或协议变化时显示错误。

## MVP 交互

- 悬浮球显示短期窗口剩余百分比。
- 绿色表示剩余量大于 30%，黄色表示剩余量不高于 30%，红色表示剩余量不高于 15%。
- 点击或展开后显示详细面板。
- 面板包含短期窗口、长期窗口、credits、模型用量桶和最后更新时间。
- 顶部刷新按钮用于立即更新。
- 底部预留固定位置、设置和退出按钮。

## 暂缓功能

- 开机自启。
- 系统托盘菜单。
- 拖拽位置持久化。
- 低额度系统通知。
- 低额度时提示切换模型。
- 历史用量曲线。
- 自动切换模型或自动修改 Codex 配置。

## 待确认问题

- 默认悬浮球是否只显示短期窗口，还是轮播短期窗口和长期窗口。
- 低额度阈值是否采用 30% / 15%，还是改成可配置。
- 是否需要低于阈值时弹 Windows 通知。
- 是否允许未来加入“一键打开 Codex 设置页”。
- 是否需要完全隐藏主窗口，仅保留托盘和悬浮球。
